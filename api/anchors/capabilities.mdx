## POST /api/anchors/ops

Administrative endpoint for managing the anchor catalog. Supports three actions:

1. **sync** - Import anchors and refresh capabilities (automated cron job)
2. **refresh_capabilities** - Re-check SEP capabilities for existing anchors
3. **import_directory** - Import anchors from external directory or JSON file

### Authentication

Requires one of:
- Query parameter `?secret=...` matching `CRON_SECRET` env var
- Vercel cron header `x-vercel-cron` (for scheduled jobs)
- `NODE_ENV !== "production"` (development only)

### Request body

<ParamField body="action" type="string" required>
  Operation to perform: `sync`, `refresh_capabilities`, or `import_directory`
</ParamField>

#### Common parameters

<ParamField body="limit" type="number" default="100">
  Maximum number of anchors to process (for refresh_capabilities)
</ParamField>

<ParamField body="country" type="string">
  Filter by country code for refresh_capabilities (e.g., "US")
</ParamField>

#### Import directory parameters

<ParamField body="downloadUrl" type="string">
  URL to download anchor directory JSON/CSV
</ParamField>

<ParamField body="anchors" type="array">
  Array of anchor objects to import directly (alternative to downloadUrl)
</ParamField>

<ParamField body="dryRun" type="boolean" default="true">
  If true, validate without writing to database
</ParamField>

<ParamField body="active" type="boolean" default="true">
  Mark imported anchors as active
</ParamField>

<ParamField body="allowedDomains" type="array">
  Whitelist of allowed anchor domains
</ParamField>

<ParamField body="useEnvAllowedDomains" type="boolean" default="false">
  Use `ANCHOR_DIRECTORY_ALLOWED_DOMAINS` from environment
</ParamField>

#### Sync parameters

<ParamField body="mode" type="string" default="directory">
  Discovery mode: `directory` or `horizon`
</ParamField>

<ParamField body="sourceUrl" type="string">
  Override anchor directory URL (default: `STELLAR_ANCHOR_DIRECTORY_URL` env)
</ParamField>

<ParamField body="directoryHome" type="string" default="https://anchors.stellar.org/">
  Base URL for auto-discovery
</ParamField>

<ParamField body="allowHorizon" type="boolean" default="false">
  Allow fallback to Horizon-based anchor discovery
</ParamField>

<ParamField body="refreshLimit" type="number" default="300">
  Number of anchors to refresh after import
</ParamField>

<ParamField body="sep1DisableThreshold" type="number" default="3">
  Auto-disable anchor after N consecutive SEP-1 404 errors
</ParamField>

### Response (sync)

<ResponseField name="status" type="string">
  "ok" on success
</ResponseField>

<ResponseField name="action" type="string">
  "sync"
</ResponseField>

<ResponseField name="triggeredAt" type="string">
  ISO 8601 timestamp
</ResponseField>

<ResponseField name="import" type="object">
  <ResponseField name="source" type="string">Data source URL</ResponseField>
  <ResponseField name="totalNormalized" type="number">Anchors normalized from source</ResponseField>
  <ResponseField name="written" type="number">Anchors written to database</ResponseField>
  <ResponseField name="skippedSourceRows" type="number">Invalid rows skipped</ResponseField>
</ResponseField>

<ResponseField name="refresh" type="object">
  <ResponseField name="requested" type="number">Refresh limit</ResponseField>
  <ResponseField name="processed" type="number">Anchors processed</ResponseField>
  <ResponseField name="ok" type="number">Successful refreshes</ResponseField>
  <ResponseField name="errors" type="number">Failed refreshes</ResponseField>
</ResponseField>

<ResponseField name="results" type="array">
  Per-anchor refresh results
  <Expandable title="Result object">
    <ResponseField name="id" type="string">Anchor ID</ResponseField>
    <ResponseField name="domain" type="string">Anchor domain</ResponseField>
    <ResponseField name="status" type="string">"ok" or "error"</ResponseField>
    <ResponseField name="error" type="string">Error message if failed</ResponseField>
    <ResponseField name="autoDisabled" type="boolean">Whether anchor was auto-disabled</ResponseField>
    <ResponseField name="sep1_404_count" type="number">Consecutive 404 count</ResponseField>
  </Expandable>
</ResponseField>

**401 Unauthorized**

```json

  "error": "Unauthorized cron request. Provide ?secret=... matching CRON_SECRET or configure Vercel cron header."
}
```

```json
{
  "error": "Invalid action. Use: import_directory | refresh_capabilities | sync"
```

## Use cases

### Scheduled sync (Vercel cron)

```yaml vercel.json
{
  "crons": [
    {
      "path": "/api/anchors/ops?action=sync",
      "schedule": "0 */6 * * *"
    }
  ]
}
```

### Manual capability refresh

```bash
curl -X POST https://api.payonproof.com/api/anchors/ops \
  -H "Content-Type: application/json" \
  -d '{"action": "refresh_capabilities", "country": "US", "limit": 50}' \
  "?secret=$CRON_SECRET"
```

### Import custom anchor list

```bash
curl -X POST https://api.payonproof.com/api/anchors/ops \
  -H "Content-Type: application/json" \
  -d '{
    "action": "import_directory",
    "anchors": [
      {
        "domain": "custom.anchor.com",
        "name": "Custom Anchor",
        "country": "BR",
        "currency": "BRL"
      }
    ],
    "dryRun": false
  }' \
  "?secret=$CRON_SECRET"
```

## Implementation notes

- Auto-disables anchors after `sep1DisableThreshold` consecutive SEP-1 404 errors
- Applies trust policies (requires SEP-10, signing key, SEP-24/SEP-31)
- Supports auto-discovery from anchors.stellar.org via HTML scraping and `__NEXT_DATA__` extraction
- Horizon fallback discovers anchors from asset issuers on the Stellar network
- Domain allowlists prevent importing untrusted anchors
